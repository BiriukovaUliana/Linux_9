## Завдання 9.5 (acces_test)
Напишіть програму, яка створює тимчасовий файл від імені звичайного користувача. Потім від імені суперкористувача використовує команди chown і chmod, щоб змінити тип володіння та права доступу.
Програма повинна визначити, в яких випадках вона може виконувати читання та запис файлу, використовуючи свій обліковий запис.

### Обгрунтування:
Програма створює файл test_file.txt з правами 0644. На цьому етапі користувач є власником файлу, тому має до нього повний доступ.

Для визначення можливості читання та запису використовується системна функція access(). Вона перевіряє реальні права процесу щодо конкретного файлу. Це дозволяє програмі зрозуміти, чи може вона взаємодіяти з об'єктом.

За допомогою функції system() виконуються команди chown - зміна власника на суперкористувача root та chmod - встановлення прав 600 — доступ тільки для власника. Оскільки власником стає root, звичайний користувач втрачає будь-які права на цей файл.

Після зміни власника повторний виклик access() покаже "ЗАБОРОНЕНО" для обох операцій. Це демонструє сувору ієрархію прав у Linux: навіть якщо програма сама створила файл, після передачі прав суперкористувачу вона більше не може ним керувати.

### Завдання 9.6 (dir_explorer)
Напишіть програму, яка виконує команду ls -l, щоб переглянути власника і права доступу до файлів у своєму домашньому каталозі, в /usr/bin та в /etc.
Продемонструйте, як ваша програма намагається обійти різні власники та права доступу користувачів, а також здійснює спроби читання, запису та виконання цих файлів.

###Обгрунтування:
Використання команди ls -l дозволяє побачити, що в домашньому каталозі більшість файлів належать користувачу uliana, тоді як у /usr/bin та /etc власником є root. Це наочно демонструє ієрархію володіння в Linux.

За допомогою функції access() програма намагається ініціювати операції читання, запису та виконання.

Користувач має повний доступ (RWX).

- /usr/bin/ls: програма показує, що користувач може читати та виконувати системні утиліти, але не має права їх змінювати. Це захищає систему від пошкодження коду програм.

- /etc/shadow: цей файл містить хеші паролів. Програма демонструє, що для звичайного користувача тут заблоковані всі дії. Це приклад найвищого рівня захисту конфіденційних даних.

Програма підтверджує, що операційна система Linux надійно розмежовує права. Навіть маючи доступ до термінала, користувач не може обійти права доступу до файлів інших власників без спеціальних привілеїв.

### Завдання 3 варіант
Визначте, чи можна прочитати вміст файлу, якщо всі дозволи на нього зняті. Поясніть, у яких випадках це можливо.

### Обгрунтування
У системі Linux права доступу (read, write, execute) є обов'язковими для звичайних користувачів. Якщо зняти всі дозволи, операційна система буде блокувати будь-які спроби відкрити файл.

## Винятки:

 - В ОС Linux користувач root має особливий статус. Системне ядро ігнорує стандартні біти прав доступу для цього облікового запису. Тому адміністратор може прочитати будь-який файл, навіть якщо на ньому стоять права 000. Саме тому можна переглядати вміст захищених файлів, використовуючи команду sudo.
- Якщо програма вже відкрила файл на читання до того, як права були зняті, вона зможе продовжувати читати його через уже існуючий дескриптор. Права перевіряються системою лише в момент відкриття файлу (open), а не під час кожного читання (read).
- У сучасних версіях Linux існують Capabilities. Якщо процесу надано таку специфічну можливість, він зможе ігнорувати обмеження прав доступу, навіть не будучи повноцінним root-користувачем.

Звичайний користувач ніколи не зможе прочитати файл без відповідних дозволів. Це є основою безпеки багатокористувацької системи. Проте повний доступ завжди залишається у адміністратора системи через механізм ігнорування стандартних обмежень.

## Команди, що були використані для перевірки:
touch secret.txt - створити файл
chmod 000 secret.txt - зняти всі права
cat secret.txt - отримаємо "Permission denied"
sudo cat secret.txt - прочитаємо файл
