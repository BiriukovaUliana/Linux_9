# Практична робота 9

## Завдання 9.1 (user_check)
Напишіть програму, яка читає файл /etc/passwd за допомогою команди getent passwd, щоб дізнатись, які облікові записи визначені на вашому комп’ютері.
Програма повинна визначити, чи є серед них звичайні користувачі (ідентифікатори UID повинні бути більші за 500 або 1000, залежно від вашого дистрибутива), окрім вас.

### У ході цього завдання було виконано такі вимоги:
Використання системної команди getent passwd для отримання списку користувачів.
- Фільтрація користувачів за значенням UID.
- Програмне ігнорування поточного активного користувача.

### Обгрунтування: 
- Використано модуль child_process для запуску системної команди.
- Виконано парсинг результату (розбиття рядка за роздільником :).
- Додано логічну перевірку uid >= 1000 && username !== currentUser.

Після запуску команди node user_check.js  отримано фінальний результат:

Програма успішно ідентифікувала користувача nobody (UID: 65534).
Власний обліковий запис було коректно відфільтровано, що підтверджує правильність роботи алгоритму.

## Завдання 9.2 (sudo_task)
Напишіть програму, яка виконує команду cat /etc/shadow від імені адміністратора, хоча запускається від звичайного користувача.
(Ваша програма повинна робити необхідне, виходячи з того, що конфігурація системи дозволяє отримувати адміністративний доступ за допомогою відповідної команди.)

Для виконання завдання було використано команду sudo cat /etc/shadow всередині методу exec().
Коли Node.js ініціює цей процес, операційна система перехоплює запит і вимагає автентифікації користувача.
Після успішного введення пароля, процес отримує права root, зчитує файл і передає його вміст назад у програму через стандартний потік виводу (stdout).

<img width="502" height="160" alt="image" src="https://github.com/user-attachments/assets/665fc9d7-c85b-4ef7-af07-ba5c74f0164f" />

В результаті ми бачимо структуру файлу /etc/shadow. Кожен рядок містить декілька полів, розділених двокрапкою:
- Ім'я користувача.
- Зашифрований пароль.
- Числові значення — це метадані пароля.

Програма довела свою працездатність, отримавши доступ до файлу, який зазвичай заблокований для користувача.

## Завдання 9.3 (file_rights)
Напишіть програму, яка від імені root копіює файл, який вона перед цим створила від імені звичайного користувача. Потім вона повинна помістити копію у домашній каталог звичайного користувача.
Далі, використовуючи звичайний обліковий запис, програма намагається змінити файл і зберегти зміни. Що відбудеться?
Після цього програма намагається видалити цей файл за допомогою команди rm. Що відбудеться?

### Обгрунтування:
Виконалась системна команда sudo cp, яка створює копію файлу. Оскільки операція виконується через sudo, власником нового файлу автоматично стає root.

<img width="482" height="107" alt="image" src="https://github.com/user-attachments/assets/3463f147-05ce-4e48-8324-5fdb2472e600" />

Програма намагається дописати текст від імені звичайного користувача.

Програма ініціює видалення файлу за допомогою команди rm.

Копіювання через sudo змінює власника файлу на адміністратора, що обмежує можливість редагування вмісту звичайними користувачами.

Видалення файлу є операцією модифікації списку файлів у директорії. Тому власник папки може видалити файл, який йому не належить, якщо він має права на запис у цю папку.
